RxJS TestScheduler

核心概念：TestScheduler 继承自 VirtualTimeScheduler，用于虚拟化时间，同步且确定性地测试使用计时器的异步 RxJS 代码（如 delay、debounceTime 等），支持 ASCII 大理石图断言。

类定义

class TestScheduler extends VirtualTimeScheduler { static parseMarblesAsSubscriptions(marbles: string, runMode?: boolean): SubscriptionLog static parseMarbles(marbles: string, values?: any, errorValue?: any, materializeInnerObservables?: boolean, runMode?: boolean): TestMessage[] constructor(assertDeepEqual: (actual: any, expected: any) => boolean | void) get hotObservables: HotObservable<any>[] get coldObservables: ColdObservable<any>[] assertDeepEqual: (actual: any, expected: any) => boolean | void createTime(marbles: string): number createColdObservable<T>(marbles: string, values?: object, error?: any): ColdObservable<T> createHotObservable<T>(marbles: string, values?: object, error?: any): HotObservable<T> expectObservable(observable: Observable, subscriptionMarbles?: string): { toBe: ... } expectSubscriptions(actualSubscriptionLogs: SubscriptionLog[]): { toBe: ... } flush(): void run<T>(callback: (helpers: RunHelpers) => T): T }

描述

使用 TestScheduler 虚拟化时间，可以同步且确定性地测试异步 RxJS 代码。ASCII 大理石图提供直观方式表示 Observable 行为，用于断言 Observable 是否符合预期。

目前 TestScheduler 主要适用于使用计时器的代码（使用 AsyncScheduler 且延迟 > 0），如 delay、debounceTime 等。若代码消费 Promise 或使用 AsapScheduler / AnimationFrameScheduler 调度，则无法可靠地用 TestScheduler 测试。

推荐使用 testScheduler.run(callback)：在回调中，使用计时器的操作符会自动使用 TestScheduler 的虚拟时间，无需像过去那样手动传入 scheduler。

静态方法

方法说明
parseMarblesAsSubscriptions(marbles, runMode?)解析订阅大理石图，返回 SubscriptionLog
parseMarbles(marbles, values?, errorValue?, ...)解析大理石图，返回 TestMessage[]

实例方法

方法说明
createTime(marbles)从大理石图创建时间帧，返回 number
createColdObservable(marbles, values?, error?)创建冷 Observable（订阅时才开始）
createHotObservable(marbles, values?, error?)创建热 Observable（测试开始时已在"运行"）
expectObservable(observable, subscriptionMarbles?)安排 Observable 断言，返回带 toBe 的对象
expectSubscriptions(logs)安排订阅断言，返回带 toBe 的对象
flush()执行所有排队的操作
run(callback)在虚拟时间环境中运行测试，helpers 提供 cold、hot、expectObservable 等

run() 示例

import { TestScheduler } from 'rxjs/testing'; const testScheduler = new TestScheduler((actual, expected) => { expect(actual).toEqual(expected); }); testScheduler.run(({ cold, expectObservable }) => { const source$ = cold('-a-b-c|', { a: 1, b: 2, c: 3 }); const expected = '-a-b-c|'; expectObservable(source$).toBe(expected, { a: 1, b: 2, c: 3 }); });

delay 测试示例

testScheduler.run(({ cold, expectObservable }) => { const source$ = cold('a-b|', { a: 'x', b: 'y' }); const result$ = source$.pipe(delay(20)); // a 在 frame 0 发出，delay 20 后在 frame 20 到达 expectObservable(result$).toBe('20ms a 20ms b|', { a: 'x', b: 'y' }); });
