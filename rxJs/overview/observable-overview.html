RxJS Observable 可观察对象

核心概念：可观察对象是多个值的惰性推式集合。它们填补了以下表格中的缺失点：
单 多
拉 (Pull) Function Iterator
推 (Push) Promise Observable

拉与推

拉（Pull）和推（Push）是两种不同的协议，描述了数据生产者如何与数据消费者通信。

什么是拉？

在拉式系统中，消费者决定何时从数据生产者接收数据。生产者本身并不知道何时将数据传递给消费者。

  • 每个 JavaScript 函数都是一个拉式系统
  • ES2015 生成器函数和迭代器（function*）也是拉式系统
生产者 消费者
拉 被动：在需要时产生数据 主动：决定何时请求数据
推 主动：按自己的节奏生成数据 被动：对收到的数据做出反应

什么是推？

在推送系统中，生产者决定何时将数据发送给消费者。消费者不知道何时接收该数据。

  • Promise：将已解析的值传递给已注册的回调，Promise 负责精确确定何时将该值"推送"到回调
  • Observable：多个值的生产者，将它们"推送"到观察者（消费者）

可观察对象 vs 函数

关键区别：函数只能返回单个值，而 Observable 可以随时间推移"返回"多个值。

对比示例

// 函数 - 只能返回一个值 function foo() { console.log('你好'); return 42; return 100; // 死代码，永远不会执行 } // Observable - 可以返回多个值 const foo$ = new Observable(subscriber => { console.log('你好'); subscriber.next(42); subscriber.next(100); // 可以返回另一个值 subscriber.next(200); // 还可以返回更多值 });

Observable 剖析

Observable 涉及四个核心方面：

  1. 创建：使用 new Observable 或创建操作符（如 of、from、interval 等）
  2. 订阅：使用观察者订阅 Observable
  3. 执行：Observable 传递 next / error / complete 通知给观察者
  4. 清理：取消 Observable 的执行

Observable 语法约定

next*(error|complete)? 含义： - 可以传递零次到无限次 next 通知 - 如果传递了 error 或 complete 通知，则此后无法再传递其他内容

可观察对象示例

以下是可观察到按压值 1、2、3 立即（同步）时所订阅，并且值 4 在一秒之后完成：

const observable = new Observable(subscriber => { subscriber.next(1); subscriber.next(2); subscriber.next(3); setTimeout(() => { subscriber.next(4); subscriber.complete(); }, 1000); }); console.log('订阅前'); observable.subscribe({ next(x) { console.log('收到值 ' + x); }, error(err) { console.error('发生错误: ' + err); }, complete() { console.log('完成'); } }); console.log('订阅后');

控制台输出：

just before subscribe got value 1 got value 2 got value 3 just after subscribe got value 4 done
