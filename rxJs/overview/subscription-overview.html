RxJS 订阅

核心概念：订阅是代表可抛弃资源的对象，通常是 Observable 的执行。订阅具有一个最重要的方法 unsubscribe()，它不带任何参数，只是处置该订阅所拥有的资源。

什么是订阅？

订阅是 Observable 执行时返回的对象，用于控制和管理 Observable 的生命周期。通过订阅，你可以：

  • 启动 Observable 的执行
  • 接收 Observable 发出的值
  • 取消订阅以释放资源
注意：在以前的 RxJS 版本中，订阅曾被称为 "Disposable"（一次性对象）。

基本订阅示例

import { interval } from 'rxjs'; const observable = interval(1000); const subscription = observable.subscribe(x => console.log(x)); // 5秒后取消订阅 setTimeout(() => { // 这会取消正在进行的 Observable 执行 subscription.unsubscribe(); console.log('已取消订阅'); }, 5000);

执行后，控制台会显示：

0 1 2 3 4 已取消订阅

订阅的核心方法

方法 说明 用途
unsubscribe() 取消订阅，释放资源 停止 Observable 执行，防止内存泄漏
add(subscription) 添加子订阅 将多个订阅组合在一起统一管理
remove(subscription) 移除子订阅 取消之前添加的子订阅
重要提醒：订阅实际上仅具有 unsubscribe() 一个功能，就是释放资源或取消可观察执行。

组合订阅

订阅可以放在一起，这样对一个订阅调用 unsubscribe() 可以同时取消订阅多个订阅。你可以通过将一个订阅"添加"到另一个订阅来实现：

import { interval } from 'rxjs'; const observable1 = interval(400); const observable2 = interval(300); const subscription = observable1.subscribe(x => console.log('第一个: ' + x) ); const childSubscription = observable2.subscribe(x => console.log('第二个: ' + x) ); // 将 childSubscription 添加为 subscription 的子订阅 subscription.add(childSubscription); setTimeout(() => { // 这会同时取消 subscription 和 childSubscription subscription.unsubscribe(); console.log('所有订阅已取消'); }, 1000);

执行后，控制台会显示（时间间隔不同）：

second: 0 first: 0 second: 1 first: 1 second: 2 所有订阅已取消

移除子订阅

订阅还具有一种 remove(otherSubscription) 方法，用于撤销之前添加的子订阅：

import { interval } from 'rxjs'; const observable1 = interval(1000); const observable2 = interval(500); const mainSubscription = observable1.subscribe(x => console.log('主订阅: ' + x) ); const childSubscription = observable2.subscribe(x => console.log('子订阅: ' + x) ); mainSubscription.add(childSubscription); // 3秒后移除子订阅，使其不再受主订阅管理 setTimeout(() => { mainSubscription.remove(childSubscription); console.log('子订阅已从主订阅中移除'); // 此时取消主订阅不会影响子订阅 mainSubscription.unsubscribe(); console.log('主订阅已取消，子订阅仍在运行'); }, 3000); // 5秒后取消子订阅 setTimeout(() => { childSubscription.unsubscribe(); console.log('子订阅已取消'); }, 5000);

防止内存泄漏的最佳实践

1. 及时取消订阅

import { interval } from 'rxjs'; export class MyComponent { private subscription; ngOnInit() { this.subscription = interval(1000).subscribe( x => console.log(x) ); } ngOnDestroy() { // 组件销毁时取消订阅，防止内存泄漏 this.subscription.unsubscribe(); } }

2. 使用 takeUntil 自动管理订阅

import { interval, Subject } from 'rxjs'; import { takeUntil } from 'rxjs/operators'; export class MyComponent { private destroy$ = new Subject(); ngOnInit() { interval(1000) .pipe(takeUntil(this.destroy$)) .subscribe(x => console.log(x)); } ngOnDestroy() { // 发出信号，所有使用 takeUntil 的订阅都会自动完成 this.destroy$.next(); this.destroy$.complete(); } }

3. 使用 take 限制订阅次数

import { interval } from 'rxjs'; import { take } from 'rxjs/operators'; // 自动在接收 5 个值后完成订阅 interval(1000) .pipe(take(5)) .subscribe( x => console.log(x), null, () => console.log('自动完成') );
