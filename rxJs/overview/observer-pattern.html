RxJS 观察者模式

核心概念：观察者（Observer）是 Observable 传递的值的消费者。观察者是一组回调，一个用于 Observable 可能传递的每种通知类型：next、error 和 complete。

什么是观察者？

观察者是一个具有三个回调的对象，分别对应 Observable 可能传递的三种通知类型：

const observer = { next: x => console.log('观察者收到 next 值: ' + x), error: err => console.error('观察者收到 error: ' + err), complete: () => console.log('观察者收到 complete 通知') };

要使用 Observer，请将其提供给 Observable 的 subscribe 方法：

const observer = { next: x => console.log('观察者收到 next 值: ' + x), error: err => console.error('观察者收到 error: ' + err), complete: () => console.log('观察者收到 complete 通知') }; observable.subscribe(observer);

观察者的特点

  • 完整性：完整的观察者包含 next、error、complete 三个回调
  • 可选性：观察者也可以是局部的 - 如果只提供某些回调，其他类型的通知会被忽略
  • 回调参数：订阅 Observable 时，可以直接提供回调作为参数，而不必包装在观察者对象中

局部观察者示例

以下是只有 next 和 error 回调的观察者（没有 complete 回调）：

const observer = { next: x => console.log('观察者收到 next 值: ' + x), error: err => console.error('观察者收到 error: ' + err) }; // 缺少 complete 回调，当 Observable 完成时，不会触发任何处理

简化形式 - 回调作为参数

订阅 Observable 时，可以直接提供回调作为参数，而不必附加到 Observer 对象：

// 只提供 next 回调 observable.subscribe( x => console.log('观察者收到 next 值: ' + x) );

在内部，observable.subscribe 会使用第一个回调参数作为 next 处理程序创建一个 Observer 对象。

所有三种回调类型作为参数

也可以提供所有三个回调作为参数（依次是 next、error、complete）：

observable.subscribe( x => console.log('观察者收到 next 值: ' + x), err => console.error('观察者收到 error: ' + err), () => console.log('观察者收到 complete 通知') );

观察者通知类型

通知类型 用途 触发次数
next 处理 Observable 发送的值 零次或多次
error 处理 Observable 中的错误 最多一次
complete 处理 Observable 完成事件 最多一次
