RxJS 主题

核心概念：Subject 是一种特殊类型的 Observable，它允许将值多播给多个观察者。普通 Observable 是单播的（每个观察者拥有独立的执行），而 Subject 是多播的，多个观察者共享同一个执行。

什么是 Subject？

Subject 既是 Observable 又是 Observer：

  • 作为 Observable：可以订阅它，接收它发出的值
  • 作为 Observer：可以调用 next()、error()、complete() 方法
import { Subject } from 'rxjs'; const subject = new Subject(); // 订阅者 A subject.subscribe({ next: v => console.log('观察者 A: ' + v) }); // 订阅者 B subject.subscribe({ next: v => console.log('观察者 B: ' + v) }); // Subject 作为观察者，发射值 subject.next(1); subject.next(2); // 输出: // 观察者 A: 1 // 观察者 B: 1 // 观察者 A: 2 // 观察者 B: 2

Subject 的多播特性

与普通 Observable 的区别：

import { from } from 'rxjs'; // 普通 Observable - 单播 const observable = from([1, 2, 3]); observable.subscribe(v => console.log('观察者 A: ' + v)); observable.subscribe(v => console.log('观察者 B: ' + v)); // 输出（两个独立的执行）: // 观察者 A: 1 // 观察者 A: 2 // 观察者 A: 3 // 观察者 B: 1 // 观察者 B: 2 // 观察者 B: 3 // Subject - 多播 import { Subject } from 'rxjs'; const subject = new Subject(); subject.subscribe(v => console.log('观察者 A: ' + v)); subject.subscribe(v => console.log('观察者 B: ' + v)); subject.next(1); subject.next(2); // 输出（共享同一个执行）: // 观察者 A: 1 // 观察者 B: 1 // 观察者 A: 2 // 观察者 B: 2

Subject 的核心方法

方法 说明 用途
next(value) 发射一个值给所有观察者 主动推送数据
error(err) 发射错误通知 通知所有观察者发生错误
complete() 完成通知 通知所有观察者完成
subscribe() 订阅 Subject 作为 Observable 使用
asObservable() 返回 Observable 视图 隐藏 Subject 的 Observer 接口

不同类型的 Subject

1. Subject - 基本主题

最基础的 Subject，不保存任何历史值，订阅者只能接收到订阅之后发射的值。

import { Subject } from 'rxjs'; const subject = new Subject(); subject.next(1); // 在订阅之前，观察者收不到 subject.subscribe(v => console.log('观察者: ' + v)); subject.next(2); // 观察者能收到 subject.next(3); // 观察者能收到 // 输出: // 观察者: 2 // 观察者: 3
2. BehaviorSubject - 行为主题

保存当前值的 Subject，订阅时会立即收到最新的值，需要初始值。

import { BehaviorSubject } from 'rxjs'; // 需要初始值 const subject = new BehaviorSubject(0); subject.subscribe(v => console.log('观察者 A: ' + v)); // 输出: 观察者 A: 0（立即收到当前值） subject.next(1); subject.next(2); // 新订阅者会收到当前最新值 subject.subscribe(v => console.log('观察者 B: ' + v)); // 输出: 观察者 B: 2 subject.next(3); // 输出: // 观察者 A: 3 // 观察者 B: 3 // 获取当前值 console.log(subject.getValue()); // 输出: 3
3. ReplaySubject - 重放主题

保存历史值的 Subject，新订阅者可以收到历史值，可以配置缓存数量或时间。

import { ReplaySubject } from 'rxjs'; // 缓存最近 3 个值 const subject = new ReplaySubject(3); subject.next(1); subject.next(2); subject.next(3); subject.next(4); subject.subscribe(v => console.log('观察者: ' + v)); // 输出: // 观察者: 2 // 观察者: 3 // 观察者: 4（只收到最近 3 个） // 也可以按时间窗口缓存 const subject2 = new ReplaySubject(100, 500); // 缓存 500ms 内的最多 100 个值
4. AsyncSubject - 异步主题

只在完成时发射最后一个值（和完成前的最后一个值）。

import { AsyncSubject } from 'rxjs'; const subject = new AsyncSubject(); subject.subscribe(v => console.log('观察者 A: ' + v)); subject.next(1); subject.next(2); subject.next(3); // 此时还没有输出，因为 subject 未完成 subject.subscribe(v => console.log('观察者 B: ' + v)); // 此时还没有输出 subject.complete(); // 完成时，所有观察者收到最后一个值 3 // 输出: // 观察者 A: 3 // 观察者 B: 3

Subject 对比表

类型 初始值 保存历史 订阅时立即收到 适用场景
Subject 不需要 不保存 否 简单事件通知
BehaviorSubject 需要 只保存当前值 是（最新值） 状态管理、当前值共享
ReplaySubject 不需要 保存指定数量/时间 是（历史值） 缓存、回放
AsyncSubject 不需要 只保存最后一个 完成时收到 异步操作最终结果

常见使用场景

1. 组件间通信

import { Subject } from 'rxjs'; // 创建一个事件总线 class EventBus { private subject = new Subject(); emit(event: string, data?: any) { this.subject.next({ event, data }); } on(event: string, handler: (data: any) => void) { return this.subject.subscribe(({ event: e, data }) => { if (e === event) handler(data); }); } } const bus = new EventBus(); // 组件 A 订阅事件 bus.on('user-login', user => { console.log('用户登录:', user); }); // 组件 B 发射事件 bus.emit('user-login', { name: '张三' });

2. 状态管理（使用 BehaviorSubject）

import { BehaviorSubject } from 'rxjs'; import { distinctUntilChanged } from 'rxjs/operators'; class Store { private state$ = new BehaviorSubject({ user: null, isLoggedIn: false, count: 0 }); // 获取当前状态 getState() { return this.state$.getValue(); } // 更新状态 setState(newState: any) { this.state$.next({ ...this.getState(), ...newState }); } // 选择特定状态（自动去重） select(key: string) { return this.state$.pipe( map(state => state[key]), distinctUntilChanged() ); } } const store = new Store(); // 订阅特定状态变化 store.select('count').subscribe(count => { console.log('计数变化:', count); }); // 更新状态 store.setState({ count: 1 }); store.setState({ count: 2 });

3. 数据缓存（使用 ReplaySubject）

import { ReplaySubject } from 'rxjs'; import { HttpClient } from '@angular/common/http'; class DataService { private cache = new Map(); getData(url: string) { if (!this.cache.has(url)) { // 创建 ReplaySubject 缓存结果 const subject = new ReplaySubject(1); this.cache.set(url, subject); // 发起 HTTP 请求 this.http.get(url).subscribe( data => subject.next(data), err => subject.error(err), () => subject.complete() ); } // 返回缓存的 Observable，新订阅者也能收到数据 return this.cache.get(url).asObservable(); } }

4. 将单播转为多播

import { Subject } from 'rxjs'; import { from } from 'rxjs'; // 原始 Observable const source$ = from([1, 2, 3]); // 使用 Subject 转为多播 const subject = new Subject(); // 多个订阅者订阅 Subject subject.subscribe(v => console.log('A:', v)); subject.subscribe(v => console.log('B:', v)); // Subject 订阅源 Observable source$.subscribe(subject);
