RxJS OperatorFunction

核心概念：OperatorFunction<T, R> 表示管道运算符的类型：接受 Observable<T>，返回 Observable<R>，是 pipeable 运算符的核心签名。

类型定义

interface OperatorFunction<T, R> extends UnaryFunction<Observable<T>, Observable<R>> { (source: Observable<T>): Observable<R> }

子接口

  • MonoTypeOperatorFunction<T> — 当 T 与 R 相同时（如 filter、tap）

描述

可管道运算符（pipeable operator）本质上是返回 OperatorFunction 的函数。签名形如：

<T, R>(source: Observable<T>) => Observable<R>

例如 map 可将 Observable<number> 转为 Observable<string>，其类型为 OperatorFunction<number, string>。

与 MonoTypeOperatorFunction<T>（T → T）相对，普通 OperatorFunction 允许输入输出类型不同。

例子

import { of } from 'rxjs'; import { map, filter } from 'rxjs/operators'; // map: OperatorFunction<number, string> of(1, 2, 3).pipe( filter((x): x is number => x > 0), // MonoTypeOperatorFunction map(x => String(x * 2)) // OperatorFunction<number, string> ).subscribe(x => console.log(x)); // 2 4 6（字符串）
