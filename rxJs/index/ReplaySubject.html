RxJS ReplaySubject

核心概念：ReplaySubject 是 Subject 的变体，通过在新订阅者首次订阅时重放旧值来"回放"历史值。

类定义

class ReplaySubject<T> extends Subject { constructor( _bufferSize: number = Infinity, _windowTime: number = Infinity, _timestampProvider?: TimestampProvider ) next(value: T): void }

说明

ReplaySubject 有一个内部缓冲区，用于存储指定数量的已观察值。与 Subject 一样，ReplaySubject 通过 next 方法接收值。当观察到值时，会根据构造时传入的配置将其存储一段时间。

当新的订阅者订阅 ReplaySubject 实例时，它会以先进先出（FIFO）方式同步发出缓冲区中的所有值。如果 ReplaySubject 已观察到 complete，它也会 complete；如果观察到 error，它也会 error。

两个主要配置项

  • bufferSize — 缓冲区中存储的项目数量，默认为无限
  • windowTime — 值在缓冲区中保留的时间（毫秒），超时后从缓冲区移除

两个配置可以同时存在。例如，最多缓冲 3 个值，且值不超过 2 秒：new ReplaySubject(3, 2000)。

与 BehaviorSubject 的区别

  • BehaviorSubject 类似 ReplaySubject(1)，但构造时需要初始值
  • ReplaySubject 在观察到 error 后仍会重放值，而 BehaviorSubject 不会

构造函数

参数类型描述
_bufferSizenumber订阅时重放的缓冲区大小，默认 Infinity
_windowTimenumber缓冲项保留时间（毫秒），默认 Infinity
_timestampProviderTimestampProvider提供当前时间戳的对象

代码示例

import { ReplaySubject } from 'rxjs'; const subject = new ReplaySubject(2); subject.next(1); subject.next(2); subject.next(3); subject.subscribe(v => console.log('观察者 A: ' + v)); subject.next(4); subject.subscribe(v => console.log('观察者 B: ' + v)); // 输出: // Observer A: 2 // Observer A: 3 // Observer A: 4 // Observer B: 3 // Observer B: 4
