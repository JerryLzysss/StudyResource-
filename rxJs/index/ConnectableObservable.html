RxJS ConnectableObservable

核心概念：ConnectableObservable 是一种特殊的 Observable，订阅时不会立即开始执行，只有调用 connect() 后才会向所有观察者共享同一次底层执行（多播）。

类声明

class ConnectableObservable<T> extends Observable<T> { constructor(source: Observable<T>, subjectFactory: () => Subject<T>) connect(): Subscription refCount(): Observable<T> }

描述

multicast 返回一个看起来像普通 Observable 的 Observable，但在订阅时却像 Subject 一样工作。multicast 返回一个 ConnectableObservable，它只是多了 connect() 方法的 Observable。

该 connect() 方法对于准确确定共享 Observable 执行何时开始非常重要。因为 connect() 在底层执行了 source.subscribe(subject)，所以它返回 Subscription，你可以取消订阅以取消共享 Observable 的执行。

主要方法

方法说明
connect()启动共享执行，将 source 订阅到内部 Subject
refCount()返回普通 Observable，在第一个订阅者到达时自动 connect，最后一个离开时断开

引用计数 refCount

若希望避免显式调用 connect()，可使用 ConnectableObservable 的 refCount() 方法。当订阅者数量从 0 变为 1 时自动 connect；从 1 变为 0 时完全取消订阅，停止进一步执行。

refCount() 仅存在于 ConnectableObservable 上，并且返回 Observable，而不是另一个 ConnectableObservable。

例子

使用 multicast 与 connect

import { from, Subject } from 'rxjs'; import { multicast } from 'rxjs/operators'; const source = from([1, 2, 3]); const subject = new Subject(); const multicasted = source.pipe(multicast(subject)); multicasted.subscribe({ next: v => console.log(`observerA: ${v}`) }); multicasted.subscribe({ next: v => console.log(`observerB: ${v}`) }); multicasted.connect();

使用 refCount 自动连接

import { interval, Subject } from 'rxjs'; import { multicast, refCount } from 'rxjs/operators'; const source = interval(500); const subject = new Subject(); const refCounted = source.pipe(multicast(subject), refCount()); refCounted.subscribe({ next: v => console.log(`observerA: ${v}`) }); setTimeout(() => { refCounted.subscribe({ next: v => console.log(`observerB: ${v}`) }); }, 600);
