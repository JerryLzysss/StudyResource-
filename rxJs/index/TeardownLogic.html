RxJS TeardownLogic

核心概念：TeardownLogic 是清理逻辑的类型别名，表示订阅取消时可执行的资源释放方式。

类型定义

type TeardownLogic = | Subscription | Unsubscribable | (() => void) | void;

描述

TeardownLogic 用于描述 Observable 订阅函数（subscribe）的返回值，以及 Subscription.add() 可接受的清理逻辑。当订阅被取消（unsubscribe）时，这些清理逻辑会被执行。

可以是以下四种形式之一：

  • Subscription — 子订阅，取消时自动 unsubscribe
  • Unsubscribable — 具有 unsubscribe 方法的对象
  • (() => void) — 清理函数，取消时调用
  • void — 无清理逻辑

使用场景

场景说明
Observable 构造函数subscribe 函数返回 TeardownLogic，在 unsubscribe 时执行
Subscription.add()将 TeardownLogic 添加为 finalizer，父订阅取消时一并清理
fromEvent 等返回移除事件监听器的函数作为 TeardownLogic

例子

import { Observable } from 'rxjs'; const custom$ = new Observable(subscriber => { const id = setInterval(() => subscriber.next(Date.now()), 1000); // 返回清理函数作为 TeardownLogic return () => { clearInterval(id); console.log('已清理 interval'); }; }); const sub = custom$.subscribe(v => console.log(v)); // 取消订阅时自动执行清理函数 sub.unsubscribe();
