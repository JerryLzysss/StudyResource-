RxJS bindNodeCallback

核心概念：bindNodeCallback 将 Node.js 样式的回调 API 转换为返回 Observable 的函数。回调应遵循 callback(error, result) 约定。

签名

bindNodeCallback<T>( callbackFunc: Function, resultSelector?: Function | SchedulerLike, scheduler?: SchedulerLike ): (...args: any[]) => Observable<T>

描述

就像 bindCallback，但是回调应该是 type callback(error, result)。

bindNodeCallback 不是运算符，因为其输入和输出都不是 Observable。输入是带有若干参数的函数 func，最后一个参数必须是完成后调用的回调函数。回调函数应遵循 Node.js 约定，其中回调的第一个参数是错误对象，表示调用是否成功。

当使用参数调用输出函数时，它将返回一个 Observable：

  • 若 func 用错误参数调用其回调，Observable 也将以该值发生错误
  • 若未传递错误参数，Observable 将发出第二个参数
  • 若有更多参数，Observable 将发出除第一个 error 参数以外的所有参数组成的数组

请注意，func 在订阅结果 Observable 时才被调用。默认同步执行，可通过 scheduler 参数更改。SchedulerLike 工作原理与 bindCallback 完全相同。

与中一样 bindCallback，输入函数的 this 上下文在被调用时将设置为返回函数的上下文。

Observable 发出值后立即完成。如需处理多次调用回调的函数，请使用 fromEvent 或 fromEventPattern。

例子

在具有多个参数的函数调用回调上使用

someFunction((err, a, b) => { console.log(err); // null console.log(a); // 5 console.log(b); // "some string" }); const boundSomeFunction = bindNodeCallback(someFunction); boundSomeFunction().subscribe(value => { console.log(value); // [5, "some string"] });

用于常规样式的函数调用回调（会出错）

someFunction(a => { console.log(a); // 5 }); const boundSomeFunction = bindNodeCallback(someFunction); boundSomeFunction().subscribe( value => {}, // 永远不会被调用 err => console.log(err) // 5 );
