RxJS SchedulerAction

核心概念：SchedulerAction 是调度程序执行的工作单元，继承自 Subscription，表示一次被调度的任务。

接口定义

interface SchedulerAction<T> extends Subscription { schedule(state?: T, delay?: number): Subscription }

描述

SchedulerAction 代表调度程序上的一次具体执行动作。当调用 SchedulerLike.schedule() 时，会创建一个 SchedulerAction 实例来执行传入的 work 函数。

由于 SchedulerAction 继承 Subscription，可以通过 unsubscribe() 取消尚未执行或正在执行的任务。在 work 函数内部，this 指向当前 SchedulerAction，可用于递归调度（如 interval 任务）。

方法

方法说明
schedule(state?, delay?)重新调度此 Action，可选传入新状态和延迟，返回 Subscription
unsubscribe()取消此 Action（继承自 Subscription）

例子

import { asyncScheduler } from 'rxjs'; function task(state) { console.log(state); this.schedule(state + 1, 1000); // `this` 是当前 SchedulerAction， // 用新状态和延迟重新调度 } asyncScheduler.schedule(task, 1000, 0); // 输出: // 0（1 秒后） // 1（2 秒后） // 2（3 秒后） // ...
