RxJS Subscriber

核心概念：Subscriber 实现了 Observer 接口并继承 Subscription 类。所有 Observer 在内部都会转换为 Subscriber，以提供 unsubscribe 等 Subscription 能力。

类定义

class Subscriber<T> extends Subscription implements Observer<T> { static create(next?, error?, complete?): Subscriber<T> constructor(destination?: Subscriber | Observer) protected isStopped: boolean protected destination: Subscriber | Observer next(value: T): void error(err?: any): void complete(): void unsubscribe(): void protected _next(value: T): void protected _error(err: any): void protected _complete(): void }

描述

Observer 是消费 Observable 值的公共 API，但所有 Observer 都会被转换为 Subscriber，以便提供类似 Subscription 的功能（如 unsubscribe）。

Subscriber 是 RxJS 中的常见类型，对实现运算符至关重要，但作为公共 API 很少直接使用。自定义运算符时，通常会创建或扩展 Subscriber。

静态方法

方法说明
create(next?, error?, complete?)根据（可能部分的）Observer 定义静态创建 Subscriber

实例方法

方法说明
next(value)接收 next 类型通知
error(err)接收 error 类型通知
complete()接收 complete 类型通知
unsubscribe()取消订阅并释放资源
_next / _error / _complete受保护的内部转发方法，子类可重写

例子

import { Subscriber, of } from 'rxjs'; // 使用 Subscriber.create 创建 const sub = Subscriber.create( v => console.log('next:', v), err => console.error('error:', err), () => console.log('完成') ); of(1, 2, 3).subscribe(sub); // 输出: // next: 1 // next: 2 // next: 3 // complete
