RxJS bindCallback

核心概念：bindCallback 将接受回调作为最后一个参数的函数，转换为返回 Observable 的函数。它不是运算符，而是创建函数。

签名

bindCallback<T>( callbackFunc: Function, resultSelector?: Function | SchedulerLike, scheduler?: SchedulerLike ): (...args: any[]) => Observable<T>

描述

给它一个类型为 f(x, callback) 的函数，它将返回一个函数 g，当以 g(x) 调用时会输出一个 Observable。

bindCallback 不是运算符，因为其输入和输出都不是 Observable。输入是带有若干参数的函数 func，最后一个参数必须是 func 完成时调用的回调函数。

输出是 bindCallback 返回的函数，参数与 func 相同（不含回调）。调用该函数时返回 Observable：

  • 若 func 用单个参数调用回调，Observable 发出该值
  • 若回调被多个参数调用，Observable 发出包含这些值的数组

请注意，func 不会在调用输出函数时立即执行，而是在订阅结果 Observable 时调用。默认情况下，对 func 的调用在订阅后同步发生，但可通过可选的第三个参数 scheduler 更改。SchedulerLike 还可控制 Observable 何时发出回调中的值。

Observable 发出值后立即完成。即使 func 再次调用回调，第二次及后续调用的值也不会出现在流中。如需处理多次回调的函数，请使用 fromEvent 或 fromEventPattern。

如果回调遵循 Node.js 约定（第一个参数为 error），请使用 bindNodeCallback 代替。

例子

将回调函数转换为 Observable

import { bindCallback } from 'rxjs'; function fetchData(id, callback) { setTimeout(() => callback(`数据-${id}`), 500); } const fetchData$ = bindCallback(fetchData); fetchData$(42).subscribe(x => console.log(x)); // 500ms 后输出: "数据-42"

回调返回多个参数

import { bindCallback } from 'rxjs'; function compute(a, b, callback) { callback(a + b, a * b); } const compute$ = bindCallback(compute); compute$(3, 4).subscribe(value => console.log(value)); // 输出: [7, 12]
