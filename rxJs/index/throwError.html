RxJS throwError

核心概念：throwError 创建一个 Observable，不向观察者发出任何项目，仅发出错误通知。

签名

throwError(error: any, scheduler?: SchedulerLike): Observable<never>

参量

参数说明
error要传递给错误通知的特定错误（RxJS 7+ 推荐传入工厂函数 () => new Error('...')）
scheduler可选。用于调度错误通知发出的 SchedulerLike

描述

只是发出"错误"，别无其他。此静态创建函数可用于创建仅发出错误通知的 Observable，它可以用来与其他可观测值组合，例如在 mergeMap 中。

例子

发出数字 7，然后发出错误

import { throwError, concat, of } from 'rxjs'; const result = concat(of(7), throwError(() => new Error('oops!'))); result.subscribe({ next: x => console.log(x), error: e => console.error(e) }); // 输出： // 7 // Error: oops!

将数字映射并展平到序列 'a'、'b'、'c'，但会在 2 时抛出错误

import { throwError, interval, of } from 'rxjs'; import { mergeMap } from 'rxjs/operators'; interval(1000).pipe( mergeMap(x => x === 2 ? throwError(() => 'Twos are bad') : of('a', 'b', 'c') ), ).subscribe({ next: x => console.log(x), error: e => console.error(e) });
