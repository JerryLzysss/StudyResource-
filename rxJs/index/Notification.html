RxJS Notification

核心概念：Notification 代表 Observable 可能发出的推送事件或值，并用元数据标注其类型（next、error 或 complete）。常用于 materialize、dematerialize 等操作符。

类声明

class Notification<T> { static createNext<T>(value: T): Notification<T> static createError(err?: any): Notification<never> & ErrorNotification static createComplete(): Notification<never> & CompleteNotification constructor(kind: 'N' | 'E' | 'C', value?: T, error?: any) observe(observer: PartialObserver<T>): void do(next: (value: T) => void, error?: (err: any) => void, complete?: () => void): void accept(nextOrObserver: PartialObserver<T> | ((value: T) => void), error?, complete?): void toObservable(): Observable<T> }

属性

属性类型说明
kind'N' | 'E' | 'C'通知类型：Next / Error / Complete
valueTnext 类型的值
erroranyerror 类型的错误
hasValueboolean是否为 next 类型（kind === 'N'）

描述

Notification 将 next、error、complete 三种通知包装为可传递的对象。配合 materialize 可将 Observable 的所有通知转为 next 流；dematerialize 则反向还原。

推荐使用字符串字面量 'N'、'E'、'C' 表示类型，而非已弃用的 NotificationKind 枚举。

例子

materialize 包装通知

import { of } from 'rxjs'; import { materialize, map } from 'rxjs/operators'; const letters = of('a', 'b', 13, 'd'); const upperCase = letters.pipe(map(x => x.toUpperCase())); const materialized = upperCase.pipe(materialize()); materialized.subscribe(x => console.log(x)); // Notification { kind: 'N', value: 'A', ... } // Notification { kind: 'N', value: 'B', ... } // Notification { kind: 'E', error: TypeError, ... }

dematerialize 还原通知

import { of, Notification } from 'rxjs'; import { dematerialize } from 'rxjs/operators'; const notifA = Notification.createNext('A'); const notifB = Notification.createNext('B'); of(notifA, notifB).pipe(dematerialize()).subscribe(console.log); // A, B
