RxJS onErrorResumeNext

核心概念：当任何提供的 Observable 发出 complete 或 error 通知时，立即订阅下一个源；忽略错误，继续执行后续 Observable 序列。

签名

onErrorResumeNext<T, R>(...sources: any[]): Observable<R>

参量

  • sources：Observable（或任何可观察对象），可直接传入或作为数组传入

返回值

Observable：将所有源一个接一个串联的 Observable，忽略所有错误，任何错误都会导致它继续移动到下一个源。

描述

不管发生什么，都执行一系列 Observable，即使这意味着"吞掉"错误。

onErrorResumeNext 将依次订阅它提供的每个可观察源。如果其订阅的源发出错误或完成，它将无误地移至下一个源。

如果 onErrorResumeNext 未提供参数或传入空数组，则返回 EMPTY。

它基本上是 concat，即使其中一个来源发出错误，也只会继续。注意：无法通过 onErrorResumeNext 的结果来处理源抛出的错误；若需处理，可先用 catchError 再传入。

例子

映射失败后订阅下一个 Observable

import { onErrorResumeNext, of } from 'rxjs'; import { map } from 'rxjs/operators'; onErrorResumeNext( of(1, 2, 3, 0).pipe( map(x => { if (x === 0) throw Error(); return 10 / x; }) ), of(1, 2, 3), ).subscribe( val => console.log(val), err => console.log(err), // 永远不会被调用 () => console.log('完成'), ); // 输出: // 10 // 5 // 3.3333333333333335 // 1 // 2 // 3 // done
