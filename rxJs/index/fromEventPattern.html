RxJS fromEventPattern

核心概念：fromEventPattern 从任意支持注册事件处理程序的 API 创建一个 Observable，比 fromEvent 更灵活，可处理自定义事件源。

签名

fromEventPattern<T>( addHandler: (handler: NodeEventHandler) => any, removeHandler?: (handler: NodeEventHandler, signal?: any) => void, resultSelector?: (...args: any[]) => T ): Observable<T>

参量

  • addHandler：以 handler 函数为参数，将其附加到实际事件源的函数
  • removeHandler（可选）：以 handler 函数为参数，将其从事件源中删除；若 addHandler 返回令牌，removeHandler 会将其作为第二个参数
  • resultSelector（可选）：投影函数，接受事件处理程序的所有参数并返回要发出的值

描述

fromEventPattern 允许您将任何支持注册事件处理程序功能的 API 转换为 Observable。它类似于 fromEvent，但更加灵活。实际上，fromEvent 的所有用例都可以由 fromEventPattern 处理（尽管写法更详细）。

该函数接受 addHandler 作为第一个参数，订阅返回的 Observable 时才会调用它，因此调用 fromEventPattern 时不一定立即注册处理程序。

注册后，每次监听的事件发生时，Observable 都会发出事件处理程序被调用时的第一个参数。若事件处理程序被多个参数调用，第二个及后续参数默认不会出现在 Observable 中。

若 API 支持注销事件处理程序，可将 removeHandler 作为第二个参数传入；当消费者取消订阅时，removeHandler 会被调用。

例子

发出 DOM 文档上发生的点击

import { fromEventPattern } from 'rxjs'; function addClickHandler(handler) { document.addEventListener('click', handler); } function removeClickHandler(handler) { document.removeEventListener('click', handler); } const clicks = fromEventPattern(addClickHandler, removeClickHandler); clicks.subscribe(x => console.log(x)); // 在浏览器任意位置点击，将记录 DOM MouseEvent 对象

与返回取消令牌的 API 一起使用

import { fromEventPattern } from 'rxjs'; const someAPIObservable = fromEventPattern( function(handler) { return someAPI.registerEventHandler(handler); }, function(handler, token) { someAPI.unregisterEventHandler(token); } );

与投影函数一起使用

const someAPIObservable = fromEventPattern( handler => someAPI.registerEventHandler(handler), handler => someAPI.unregisterEventHandler(handler), (eventType, eventMessage) => eventType + ' --- ' + eventMessage ); // 输出: "EVENT_TYPE --- EVENT_MESSAGE"
