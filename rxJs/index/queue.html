RxJS queue

核心概念：queue 是队列调度程序（queueScheduler），将每个任务放入队列而非立即执行，递归调度时避免栈溢出，类似蹦床（trampoline）机制。

声明

const queue: QueueScheduler;

描述

将每个下一个任务放在队列中，而不是立即执行。

queue 与 delay 一起使用时，行为与 async 调度程序相同。

无 delay 时，同步安排给定任务——调度后立即执行。但在递归调用时（在已调度任务内部再次 schedule），新任务会入队等待当前任务完成，而非立即执行。

使用 queue 调度程序时，可确保当前任务在该调度器上的其他任务开始前结束。

例子

先递归调度，再执行后续代码

import { queueScheduler } from 'rxjs'; queueScheduler.schedule(() => { queueScheduler.schedule(() => console.log('第二个任务')); console.log('第一个任务'); }); // 输出: // first // second

递归重新安排自身

import { queueScheduler } from 'rxjs'; queueScheduler.schedule(function(state) { if (state !== 0) { console.log('之前', state); this.schedule(state - 1); console.log('之后', state); } }, 0, 3); // queue 调度器输出（深度优先，当前任务完成后才执行队列中下一个）: // before 3 // after 3 // before 2 // after 2 // before 1 // after 1
