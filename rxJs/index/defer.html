RxJS defer

核心概念：defer 创建一个 Observable，在订阅时才调用 Observable 工厂函数，为每个新的 Observer 创建独立的 Observable 执行。

签名

defer<R extends ObservableInput<any> | void>( observableFactory: () => R ): Observable<ObservedValueOf<R>>

参量

参数说明
observableFactoryObservable 工厂函数，为每个订阅的 Observer 调用。也可能返回 Promise，将即时转换为 Observable

返回值

Observable<ObservedValueOf<R>>：Observers 的订阅触发对给定 Observable 工厂函数的调用。

描述

延迟创建 Observable，即仅在订阅时创建。

defer 允许您仅在观察者订阅时创建 Observable，并为每个观察者创建一个新的 Observable。它等待观察者订阅后，再生成 Observable（通常通过工厂函数）。它为每个订阅者重复此过程，因此尽管每个订阅者可能认为自己订阅的是同一个 Observable，实际上每个订阅者都拥有自己的个人 Observable。

例子

随机订阅点击或 interval

import { defer, fromEvent, interval } from 'rxjs'; const clicksOrInterval = defer(function () { return Math.random() > 0.5 ? fromEvent(document, 'click') : interval(1000); }); clicksOrInterval.subscribe(x => console.log(x)); // 若 Math.random() > 0.5，监听 document 点击并输出 MouseEvent // 否则每秒发出递增数字
