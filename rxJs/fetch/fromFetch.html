RxJS fromFetch

核心概念：fromFetch 使用浏览器原生 Fetch API 发出 HTTP 请求，返回一个在订阅时执行请求的 Observable，并通过内部 AbortController 支持取消。

函数签名

fromFetch( input: string | Request, initWithSelector: RequestInit & { selector?: (response: Response) => any } = {} ): Observable<Response | T>

参量

参数说明
input要获取的资源，可以是 URL 字符串或 Request 对象
initWithSelector可选，默认 {}。类型为 RequestInit & { selector?: (response: Response) => any }

返回值

返回 Observable：订阅时使用原生 fetch 执行 HTTP 请求，并将 Subscription 与 AbortController 绑定以支持取消。

描述

警告：部分 Fetch API 仍处于试验阶段，AbortController 需浏览器支持才能正常运行并正确使用取消功能。

fromFetch 会自动设置内部 AbortController，在订阅取消时中止内部 fetch。若通过 init 参数提供了 signal，其行为将与通常的 fetch 一致；若提供的 signal 被中止，fetch 的 Promise 通常会 reject，此时会被视为 Observable 错误。

基本用法

import { of } from 'rxjs'; import { fromFetch } from 'rxjs/fetch'; import { switchMap, catchError } from 'rxjs/operators'; const data$ = fromFetch('https://api.github.com/users?per_page=5').pipe( switchMap(response => { if (response.ok) { return response.json(); } else { return of({ error: true, message: `Error ${response.status}` }); } }), catchError(err => { console.error(err); return of({ error: true, message: err.message }); }) ); data$.subscribe({ next: result => console.log(result), complete: () => console.log('完成') });

与分块传输编码一起使用

对于使用 HTTP 响应分块传输编码的情况，fetch 返回的 Promise 将在响应头被解析后立即 resolve。这意味着 fromFetch 可观察对象会发出 Response 然后完成——在收到响应体之前。当调用 Response 的 text() 或 json() 等方法时，返回的 Promise 要等到收到完整响应体才会 resolve。取消订阅使用 Promise 作为输入的操作符不会中止请求。

为便于中止分块传输编码响应的检索，可通过 init 参数指定 selector：

import { fromFetch } from 'rxjs/fetch'; const data$ = fromFetch('https://api.github.com/users?per_page=5', { selector: response => response.json() }); data$.subscribe({ next: result => console.log(result), complete: () => console.log('完成') });
