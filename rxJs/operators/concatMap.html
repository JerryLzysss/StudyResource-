RxJS concatMap

核心概念：concatMap 将源 Observable 的每个值映射为内部 Observable，然后按顺序订阅并发出内部 Observable 的值。等价于 map + concatAll。

签名

concatMap<T, R>( project: (value: T, index: number) => ObservableInput<R>, resultSelector?: (outerValue: T, innerValue: R, outerIndex: number, innerIndex: number) => any ): OperatorFunction<T, R>

描述

每当源 Observable 发出一个值，project 函数会将其映射为一个内部 Observable。concatMap 会等待当前内部 Observable 完成后，才处理源发出的下一个值。这保证了内部 Observable 按顺序执行，不会并发。

与 mergeMap 不同，concatMap 不会并发处理多个内部 Observable；与 switchMap 不同，它不会取消未完成的内部 Observable。

例子

import { of } from 'rxjs'; import { concatMap } from 'rxjs/operators'; const source = of('你好', '再见'); const example = source.pipe( concatMap(val => of(`${val} World!`)) ); example.subscribe(val => console.log(val)); // 输出: // 你好世界！ // Goodbye World!
