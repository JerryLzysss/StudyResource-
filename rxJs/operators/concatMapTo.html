RxJS concatMapTo

核心概念：concatMapTo 在源 Observable 每次发出值时，总是订阅同一个内部 Observable，等其完成后合并结果，再处理下一个源值。

签名

concatMapTo<T, R>( innerObservable: ObservableInput<R>, resultSelector?: (outerValue: T, innerValue: R, outerIndex: number, innerIndex: number) => any ): OperatorFunction<T, R>

描述

concatMapTo 是 concatMap 的简化版，不需要 project 函数，而是始终映射到同一个内部 Observable。每当源发出一个值，就订阅该内部 Observable 并等待其完成，然后才处理源的下一次发出。

注意：如果源 Observable 发出值的速度快于内部 Observable 完成的速度，可能导致内存问题，因为未完成的内部 Observable 会被排队等待。

例子

import { interval, of } from 'rxjs'; import { concatMapTo, take } from 'rxjs/operators'; const source = interval(1000).pipe(take(3)); const message = of('秒已过!'); source.pipe( concatMapTo(message, (time, msg) => `${time} ${msg}`) ).subscribe(x => console.log(x)); // 输出: // 0 秒已过! // 1 秒已过! // 2 秒已过!
