RxJS exhaustMap

核心概念：exhaustMap 将每个源值映射为内部 Observable，但在当前内部 Observable 完成前忽略所有新的源值。常用于防止重复提交（如登录请求）。

签名

exhaustMap<T, R>( project: (value: T, index: number) => ObservableInput<R>, resultSelector?: (outerValue: T, innerValue: R, outerIndex: number, innerIndex: number) => any ): OperatorFunction<T, R>

描述

exhaustMap 等价于 map + exhaustAll。每当源发出一个值，如果当前没有正在执行的内部 Observable，则映射并订阅；否则忽略该源值。

对比其他高阶映射运算符：concatMap 会排队等待；switchMap 会取消当前内部 Observable 并切换；mergeMap 会并发执行所有内部 Observable。

例子

import { fromEvent, interval, of } from 'rxjs'; import { exhaustMap, take } from 'rxjs/operators'; fromEvent(document, 'click').pipe( exhaustMap(() => interval(1000).pipe(take(5))) ).subscribe(x => console.log(x)); // 点击后启动 interval，在 interval 完成前再次点击会被忽略 // 输出: 0, 1, 2, 3, 4
