RxJS multicast

核心概念：multicast 将冷 Observable 转换为 ConnectableObservable，使多个订阅者共享同一底层订阅，避免重复执行副作用。

签名

multicast<T>(subjectOrSubjectFactory: Subject<T> | (() => Subject<T>)): MonoTypeOperatorFunction<T>

参量

  • subjectOrSubjectFactory：Subject 实例，或返回 Subject 的工厂函数

返回值

返回 MonoTypeOperatorFunction，应用后得到 ConnectableObservable，需调用 connect() 才开始向源订阅。

描述

multicast 是创建可连接 Observable 的底层运算符。通常与 refCount() 组合，或直接使用更高级的 share()、publish() 等。

典型场景：HTTP 请求、WebSocket 连接等昂贵副作用只执行一次，多个组件共享同一数据流。

例子

import { interval, Subject } from 'rxjs'; import { multicast, tap } from 'rxjs/operators'; const source = interval(1000).pipe( tap(() => console.log('副作用执行')), multicast(() => new Subject()) ); source.subscribe(v => console.log('A:', v)); source.subscribe(v => console.log('B:', v)); const sub = source.connect(); // connect 后两个订阅者共享同一 interval
