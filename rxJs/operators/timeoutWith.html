RxJS timeoutWith

核心概念：timeoutWith 在超时时不抛出错误，而是切换到备用的 Observable 继续发出值。

签名

timeoutWith<T, R>( due: number | Date, withObservable: ObservableInput<R>, scheduler?: SchedulerLike ): OperatorFunction<T, T | R>

参量

  • due：超时时间（毫秒）或 Date
  • withObservable：超时后切换到的备用 Observable

返回值

返回 OperatorFunction，超时时切换到备用源。

描述

与 timeout 不同，timeoutWith 提供 graceful fallback，如请求超时返回缓存数据。

例子

import { timer, of } from 'rxjs'; import { timeoutWith } from 'rxjs/operators'; timer(3000).pipe( timeoutWith(1000, of('备用数据')) ).subscribe(console.log); // 1 秒后超时，输出: '备用数据'
