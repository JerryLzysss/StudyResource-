RxJS auditTime

核心概念：忽略源值 duration 毫秒，然后发出来自源 Observable 的最新值，并重复此过程。

签名

auditTime<T>( duration: number, scheduler: SchedulerLike = async ): MonoTypeOperatorFunction<T>

参量

参量说明
duration发出最新值之前需要等待的时间，以毫秒为单位（或由可选 scheduler 确定的时间单位）
scheduler可选，默认 async，用于管理限速行为的计时器

返回值

返回 MonoTypeOperatorFunction：对源 Observable 执行速率限制的 Observable。

描述

看到源值时，它将忽略接下来 duration 毫秒内的值，然后发出来自源的最新值。

auditTime 与 throttleTime 相似，但从静音时间窗口中发出最后一个值，而不是第一个值。最初计时器被禁用，最早的源值到达后计时器启用；duration 毫秒过去后，禁用计时器并在输出 Observable 上发出最新值，此过程对下一个源值重复。可选的 SchedulerLike 用于管理计时器。

例子

以每秒最多一次点击的速度发出点击

import { fromEvent } from 'rxjs'; import { auditTime } from 'rxjs/operators'; const clicks = fromEvent(document, 'click'); const result = clicks.pipe(auditTime(1000)); result.subscribe(x => console.log(x));
