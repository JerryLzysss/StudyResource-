RxJS dematerialize

核心概念：dematerialize 将 Notification 对象还原为实际的 next、error、complete 通知，是 materialize 的逆操作。

签名

dematerialize<T>(): OperatorFunction<Notification<T>, T>

描述

dematerialize 接收一个发出 Notification 对象的 Observable，将其中的 Notification 还原为对应的 Observable 通知（next、error 或 complete）。

常与 materialize 配合使用：materialize 将 Observable 的通知包装为 Notification 对象，dematerialize 则将其还原。适用于需要对 next/error/complete 进行统一处理的场景。

例子

import { of, Notification } from 'rxjs'; import { dematerialize } from 'rxjs/operators'; of( Notification.createNext('SUCCESS'), Notification.createError('ERROR!') ).pipe( dematerialize() ).subscribe({ next: val => console.log('NEXT:', val), error: err => console.log('ERROR:', err) }); // 输出: // NEXT: SUCCESS // ERROR: ERROR!
