RxJS exhaust（exhaustAll）

核心概念：exhaust（现名 exhaustAll）用于高阶 Observable，在当前内部 Observable 未完成时忽略源发出的新内部 Observable，等当前完成后才订阅下一个。

签名

exhaustAll(): OperatorFunction<ObservableInput<T>, T> // RxJS 7+ 中 exhaust 是 exhaustAll 的别名

描述

exhaustAll 订阅源 Observable 发出的内部 Observable。如果当前有内部 Observable 正在执行，则忽略源新发出的内部 Observable。只有当当前内部 Observable 完成后，才会订阅源下一个发出的内部 Observable。

与 concatAll 不同，exhaustAll 不会排队等待，而是直接丢弃被忽略的 Observable。与 mergeAll 不同，它不会并发执行多个内部 Observable。

例子

import { interval, of } from 'rxjs'; import { exhaustAll, map, take } from 'rxjs/operators'; const source = interval(1000).pipe( take(5), map(i => of(i, i + 0.1).pipe( // 模拟较慢的内部 Observable )) ); // 当前内部 Observable 未完成时，新的会被忽略
