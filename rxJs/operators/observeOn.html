RxJS observeOn

核心概念：observeOn 使用指定的 Scheduler 重新调度 Observable 的通知（next/error/complete），控制观察者在哪个执行上下文接收值。

签名

observeOn<T>(scheduler: SchedulerLike, delay?: number): MonoTypeOperatorFunction<T>

参量

  • scheduler：SchedulerLike，如 asyncScheduler、asapScheduler、queueScheduler
  • delay（可选）：每次通知之间的最小延迟（毫秒）

返回值

返回 MonoTypeOperatorFunction，通知会在指定 Scheduler 上异步投递给观察者。

描述

observeOn 只影响通知的投递时机，不改变源 Observable 的订阅时机（订阅时机由 subscribeOn 控制）。

常用于将计算结果切换到 UI 线程，或避免在特定 Scheduler 上阻塞。

例子

import { of, asyncScheduler } from 'rxjs'; import { observeOn } from 'rxjs/operators'; of(1, 2, 3).pipe( observeOn(asyncScheduler) ).subscribe(v => console.log(v)); // 通知在 asyncScheduler 上异步投递
