RxJS distinctUntilKeyChanged

核心概念：distinctUntilKeyChanged 只有指定属性的值与上一个不同时才发出，是 distinctUntilChanged 针对对象属性的简化版。

签名

distinctUntilKeyChanged<T, K extends keyof T>( key: K, compare?: (x: T[K], y: T[K]) => boolean ): MonoTypeOperatorFunction<T>

描述

distinctUntilKeyChanged 比较源 Observable 发出的对象中指定 key 的值。只有当该 key 的值与上一个对象不同时，才发出当前对象。

等价于 distinctUntilChanged((prev, curr) => prev[key] === curr[key])，但写法更简洁。常用于对象流中只关心某个字段是否变化的场景。

例子

import { from } from 'rxjs'; import { distinctUntilKeyChanged } from 'rxjs/operators'; from([ { id: 1, name: 'Alice' }, { id: 1, name: 'Alice Jr' }, { id: 2, name: 'Bob' }, { id: 2, name: 'Robert' } ]).pipe( distinctUntilKeyChanged('id') ).subscribe(x => console.log(x)); // 输出: // { id: 1, name: 'Alice' } // { id: 2, name: 'Bob' }
