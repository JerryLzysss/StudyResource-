RxJS audit

核心概念：在由另一个 Observable 确定的持续时间内忽略源值，沉默期结束后发出源 Observable 的最新值，然后重复此过程。

签名

audit<T>( durationSelector: (value: T) => SubscribableOrPromise<any> ): MonoTypeOperatorFunction<T>

参量

参量说明
durationSelector接收源 Observable 发出的值，返回用于计算沉默持续时间的 Observable 或 Promise

返回值

返回 MonoTypeOperatorFunction：对源 Observable 执行速率限制的 Observable。

描述

就像 auditTime，但沉默持续时间由第二个 Observable 确定。

audit 与 throttle 相似，但从静音时间窗口中发出最后一个值，而不是第一个值。audit 在内部计时器被禁用时，将源 Observable 的最新值发射到输出 Observable；计时器启用时则忽略源值。最初计时器被禁用，第一个源值到达后，通过 durationSelector 启用计时器并返回"持续时间" Observable；持续时间 Observable 发出一个值或完成时，禁用计时器，最近的源值在输出 Observable 上发出，此过程对下一个源值重复。

例子

以每秒最多一次的速度发出点击事件

import { fromEvent, interval } from 'rxjs'; import { audit } from 'rxjs/operators'; const clicks = fromEvent(document, 'click'); const result = clicks.pipe(audit(ev => interval(1000))); result.subscribe(x => console.log(x));
