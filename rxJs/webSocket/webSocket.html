RxJS webSocket

核心概念：webSocket 是工厂函数，用于创建 WebSocketSubject，对浏览器提供的 w3c 兼容 WebSocket 对象进行封装，既可接收消息（Observable）也可发送消息（Observer）。

函数签名

webSocket(urlConfigOrSource: string | WebSocketSubjectConfig): WebSocketSubject

参量

参数说明
urlConfigOrSourceWebSocket 端点 URL 字符串，或包含配置与生命周期观察者的 WebSocketSubjectConfig 对象

返回值

返回 WebSocketSubject，可通过 WebSocket 连接与服务器双向通信。

描述

webSocket 是工厂函数，产生 WebSocketSubject，用于与任意端点建立 WebSocket 连接。参数可以是端点 URL 字符串，也可以是 WebSocketSubjectConfig 对象，以提供额外配置及追踪连接生命周期的观察者。

接收服务器消息

import { webSocket } from 'rxjs/webSocket'; const subject = webSocket('ws://localhost:8081'); subject.subscribe({ next: msg => console.log('收到消息: ' + msg), error: err => console.log(err), complete: () => console.log('完成') });

向服务器推送消息

import { webSocket } from 'rxjs/webSocket'; const subject = webSocket('ws://localhost:8081'); subject.subscribe(); // 至少需要一个消费者订阅，否则 next 的值只会被缓冲而不会发送 subject.next({ message: 'some message' }); // 连接建立后发送消息，默认使用 JSON.stringify 序列化 subject.complete(); // 关闭连接 subject.error({ code: 4000, reason: 'I think our app just broke!' }); // 也会关闭连接，并告知服务器关闭由错误引起

多路复用（multiplex）

WebSocketSubject 支持多路复用，允许多个虚拟 socket 流共享同一个底层 WebSocket 连接。通过 multiplex 方法，可在订阅时发送订阅消息、在取消订阅时发送退订消息，并用 messageFilter 筛选服务器返回的数据。
