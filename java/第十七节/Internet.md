# 网络编程

## 软件结构

- **C/S**：全称 Client/Server，客户端与服务器结构
- **B/S**：全称 Browser/Server，浏览器与服务器结构

## 通信三要素

### IP 地址

计算机在网络中的地址标识。

- IPv4：32 位
- IPv6：128 位，扩大地址空间
- 查看本机 IP：`ipconfig`（Windows）
- 连通性测试：`ping IP`
- 本机回环：`127.0.0.1` / `localhost`

### 协议

#### TCP（面向连接）

三次握手（建立连接）：

1. 客户端向服务端发出连接请求
2. 服务端回送一个**响应**，确认收到请求
3. 客户端再次确认，连接建立

四次挥手（释放连接）：

1. 客户端提出结束连接（半关闭，通常不再发送，仍可接收）
2. 服务端处理完剩余数据后告知可以继续释放流程
3. 服务端发送释放连接的**报文**
4. 客户端回复最终断开报文；发送后可能等待 2MSL，超时重发机制用于保证可靠关闭

特点：可靠、相对安全；效率相对较低。

#### UDP（面向无连接）

特点：效率高；不保证可靠送达，可能丢包。

### 端口号

区分同一主机上的不同应用程序。可自行**指定**端口（注意勿与系统保留端口冲突）。

## UDP 编程步骤

发送端：

1. 创建 `DatagramSocket`（无参则端口随机；有参则自己指定）
2. 创建 `DatagramPacket`（数据、目标 IP、端口）
3. 发送
4. 释放资源

接收端：

1. 创建 `DatagramSocket` 并指定端口
2. 接收数据包
3. 解析数据
4. 释放资源

## TCP 编程步骤

客户端：

1. 创建 `Socket`，指定服务端 IP 与端口
2. 通过 `getInputStream` / `getOutputStream` 读写
3. 关闭流与 Socket

服务端：

1. 创建 `ServerSocket`，指定端口
2. `accept()` 等待客户端连接，返回 `Socket`
3. 通过输入输出流读写
4. 关闭资源
