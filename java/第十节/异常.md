# 异常

Java 中的异常体系大致如下：

- `Error`：严重错误（如栈溢出），一般不由业务代码捕获处理
- `Exception`：可处理的异常
  - 编译期异常（受检异常 / checked）：如 `IOException`，必须显式处理或声明抛出
  - 运行期异常（非受检异常 / unchecked）：如 `NullPointerException`，继承自 `RuntimeException`

异常处理流程：JVM 创建异常对象后，若当前方法未处理，会沿调用栈向上抛出；若一直无人处理，则打印堆栈信息并终止程序。

## 异常处理方式

### 1. throws（声明抛出）

写在方法签名的参数列表之后、方法体之前：

```java
public void read() throws IOException {
    // ...
}
```

- `throws` 表示「我可能抛出这些异常，由调用方处理」
- 可声明多个：`throws 异常1, 异常2`
- 若多个异常有继承关系，声明父类即可覆盖子类；不确定时可用 `Exception`
- 一次方法调用通常只会抛出一个异常对象

### 2. try-catch（捕获处理）

```java
try {
    // 可能出现异常的代码
} catch (异常类型 对象名) {
    // 处理异常的代码
}
```

## finally

作用：

1. 无论是否发生异常，一般都会执行（代表的是「收尾」逻辑）
2. 常用于关闭流、释放连接等资源
3. 注意：`finally` 不能替代 GC；对象内存仍由垃圾回收器管理

注意：若执行了 `System.exit(0)`，会终止 JVM，`finally` 可能不会执行。

用法：`try { ... } catch { ... } finally { ... }`

## 继承中的异常声明

- 父类方法声明了异常，子类重写时可声明相同、更小范围的异常，也可不声明
- 父类方法未声明受检异常时，子类重写一般也不应新增受检异常

## Throwable 常用方法

| 方法 | 作用 |
|------|------|
| `String toString()` | 输出异常类型和信息 |
| `String getMessage()` | 输出异常信息 |
| `void printStackTrace()` | 打印完整堆栈 |
