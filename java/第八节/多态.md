# 多态

## 两种创建方式对比

普通 `new` 子类对象：

- 优点：既能调用重写方法，也能调用父类非私有成员，还能调用子类特有成员
- 缺点：扩展性相对较差（调用处往往绑定具体子类类型）

多态方式（父类引用指向子类对象）：

- 优点：扩展性强，便于面向父类/接口编程
- 缺点：不能直接调用子类特有成员（需向下转型）

## 向上转型

父类引用指向子类对象，例如：`父类类型 obj = new 子类();`

## 向下转型

类似强制类型转换（大类型转成小类型）：

1. 向上转型：`父类类型 obj1 = new 子类();`（类似 `double b = 10;`）
2. 向下转型：`子类类型 obj2 = (子类类型) obj1;`（类似 `int a = (int) b;`）
3. 若要调用子类特有方法，可先向下转型
4. 若实际类型与目标类型不匹配，会抛出 `ClassCastException`
5. 转型前可用 `instanceof` 判断类型是否匹配
