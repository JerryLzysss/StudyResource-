# 性能

> 官方文档：[Performance | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/performance)

## 概述

Electron 应用本质上是 Chromium 浏览器加上 Node.js，因此性能优化需要同时关注 Web 性能和 Node.js 性能。

## 关键建议

### 1. 谨慎创建 BrowserWindow

每个 `BrowserWindow` 都会创建一个渲染器进程。尽量减少同时打开的窗口数量，及时销毁不需要的窗口。

### 2. 使用实用进程处理 CPU 密集型任务

将 CPU 密集型任务从主进程移到实用进程（UtilityProcess），避免阻塞 UI：

```js
const { utilityProcess } = require('electron')
const child = utilityProcess.fork('heavy-task.js')
```

### 3. 避免阻塞主进程

主进程阻塞会导致整个应用无响应。将耗时操作放到 Worker 或实用进程中。

### 4. 优化渲染器

- 使用 Chrome DevTools 分析性能
- 避免不必要的 DOM 操作
- 使用虚拟滚动处理大列表
- 懒加载非关键资源

### 5. 减少应用启动时间

- 延迟加载非必要模块
- 使用 V8 快照（snapshots）
- 优化预加载脚本大小

### 6. 内存管理

- 及时移除事件监听器
- 避免内存泄漏（特别是 IPC 监听器）
- 使用 `webContents.destroy()` 释放资源

## 测量工具

- Chrome DevTools Performance 面板
- Electron 内置的 `contentTracing` API
- `process.getProcessMemoryInfo()` 监控内存
