# 上下文隔离

> 官方文档：[Context Isolation | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/context-isolation)

## 什么是上下文隔离？

上下文隔离（Context Isolation）是一项安全功能，确保预加载脚本和 Electron 的内部逻辑在与加载到 `webContents` 中的网站**分离的上下文**中运行。这有助于防止网站访问 Electron 内部或预加载脚本拥有的强大 API。

这意味着预加载脚本访问的 `window` 对象与网站访问的 `window` 对象是**不同的**。例如，如果在预加载脚本中设置 `window.hello = 'wave'`，网站尝试访问 `window.hello` 将得到 `undefined`。

上下文隔离自 Electron 12 起默认启用，是所有应用的推荐安全设置。

## 迁移指南

### 之前：上下文隔离禁用

```js
// preload.js - 上下文隔离禁用
window.myAPI = {
  doAThing: () => {}
}
```

### 之后：上下文隔离启用

使用 `contextBridge` 模块安全地暴露 API：

```js
// preload.js - 上下文隔离启用
const { contextBridge } = require('electron')

contextBridge.exposeInMainWorld('myAPI', {
  doAThing: () => {}
})
```

## 安全注意事项

仅启用 `contextIsolation` 和使用 `contextBridge` 并不意味着一切都安全：

```js
// ❌ 不安全的代码
contextBridge.exposeInMainWorld('myAPI', {
  send: ipcRenderer.send  // 直接暴露强大 API
})
```

正确做法是每种 IPC 消息提供一个方法：

```js
// ✅ 安全的代码
contextBridge.exposeInMainWorld('myAPI', {
  loadPreferences: () => ipcRenderer.invoke('load-prefs')
})
```

## TypeScript 支持

创建 `interface.d.ts` 声明文件，扩展 `Window` 接口：

```ts
export interface IElectronAPI {
  loadPreferences: () => Promise<void>
}

declare global {
  interface Window {
    electronAPI: IElectronAPI
  }
}
```
