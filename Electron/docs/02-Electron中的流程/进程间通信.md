# 进程间通信

> 官方文档：[IPC | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/ipc)

## 概述

Electron 的主进程和渲染进程有着清楚的分工并且不可互换。解决这一问题的方法是使用**进程间通信 (IPC)**。

## 渲染器 → 主进程

### ipcRenderer.invoke / ipcMain.handle（推荐）

适用于需要返回值的双向通信：

```js
// 主进程
const { ipcMain } = require('electron')
ipcMain.handle('ping', () => 'pong')

// 预加载脚本
const { contextBridge, ipcRenderer } = require('electron')
contextBridge.exposeInMainWorld('api', {
  ping: () => ipcRenderer.invoke('ping')
})

// 渲染器
const response = await window.api.ping() // 'pong'
```

### ipcRenderer.send / ipcMain.on

适用于单向消息：

```js
// 主进程
ipcMain.on('set-title', (event, title) => {
  const webContents = event.sender
  const win = BrowserWindow.fromWebContents(webContents)
  win.setTitle(title)
})

// 预加载脚本
contextBridge.exposeInMainWorld('api', {
  setTitle: (title) => ipcRenderer.send('set-title', title)
})
```

## 主进程 → 渲染器

使用 `webContents.send` 向渲染器发送消息：

```js
// 主进程
const { BrowserWindow, ipcMain } = require('electron')

ipcMain.handle('dialog:openFile', async (event) => {
  const { canceled, filePaths } = await dialog.showOpenDialog()
  if (!canceled) {
    event.sender.send('file-selected', filePaths[0])
  }
})

// 预加载脚本
contextBridge.exposeInMainWorld('api', {
  onFileSelected: (callback) => ipcRenderer.on('file-selected', (_event, value) => callback(value))
})
```

## 安全最佳实践

1. **永远不要**通过 `contextBridge` 直接暴露 `ipcRenderer` 模块
2. 使用 `ipcMain.handle` 而非 `ipcMain.on` 处理来自渲染器的请求
3. 验证所有 IPC 消息的 `sender`
4. 为每种操作提供专用的 API 方法，而非通用的 send 函数

更多安全建议请参阅 [安全](../03-最佳实践/安全.md)。
