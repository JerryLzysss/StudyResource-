# 沙盒

> 官方文档：[Process Sandboxing | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/sandbox)

## 概述

从 Electron 20 开始，渲染器进程默认启用沙盒。沙盒化的渲染器进程只能使用有限的 API 子集。

## 预加载脚本沙盒化

沙盒化的预加载脚本只拥有一个 polyfilled 的 `require` 函数，只能访问有限的 API：

| API | 详细信息 |
|-----|----------|
| Electron | 渲染进程 API |
| Node.js | events, timers, url |
| polyfilled | Buffer, Process, clearImmediate, setImmediate |

## 启用/禁用沙盒

默认情况下沙盒是启用的。可以在 `webPreferences` 中配置：

```js
const win = new BrowserWindow({
  webPreferences: {
    sandbox: true  // 默认值
  }
})
```

> **安全建议**：强烈建议保持沙盒启用。禁用沙盒会显著降低应用安全性。

## 与上下文隔离的关系

沙盒和上下文隔离是两个独立的安全功能，建议同时启用：

- **沙盒**：限制预加载脚本可访问的 Node.js API
- **上下文隔离**：将预加载脚本的 `window` 与网页的 `window` 分离
