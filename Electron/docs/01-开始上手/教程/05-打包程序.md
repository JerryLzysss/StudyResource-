
# 打包程序

这是 Electron 教程的第 5 部分。

## Electron Forge

Electron 的核心模块中没有捆绑任何用于打包或分发文件的工具。如果在开发模式下完成了一个 Electron 应用，需要使用额外的工具来打包应用程序并分发给用户。

[Electron Forge](https://www.electronforge.io/) 是一个处理 Electron 应用程序打包与分发的一体化工具。在工具底层，它将许多现有的 Electron 工具（例如 `@electron/packager`、`@electron/osx-sign`、`electron-winstaller` 等）组合到一起。

## 导入 Forge

```sh
npm install --save-dev @electron-forge/cli
npx electron-forge import
```

导入后，`package.json` 的 `devDependencies` 下会安装更多包，并生成 `forge.config.js` 配置文件，其中包含多个 makers（每个目标平台一个）。

## 创建可分发版本

```sh
npm run make
```

`make` 命令包含两步：

1. 运行 `electron-forge package`，把应用代码与 Electron 二进制包结合起来
2. 使用打包后的文件夹为每个 maker 配置生成可分发文件

运行后会在 `out` 文件夹中看到可分发文件和源码文件夹。

自定义图标可查看 [Forge 文档](https://www.electronforge.io/guides/create-and-add-icons)。

## 代码签名

代码签名是一种可用于证明桌面应用程序是由已知来源创建的安全技术。Windows 和 macOS 拥有其特定的代码签名系统。

- **macOS**：在应用程序打包时完成代码签名
- **Windows**：对可分发文件进行签名操作

```js
// forge.config.js - Windows 示例
module.exports = {
  makers: [
    {
      name: '@electron-forge/maker-squirrel',
      config: {
        certificateFile: './cert.pfx',
        certificatePassword: process.env.CERTIFICATE_PASSWORD
      }
    }
  ]
}
```

更多分发相关内容，请参阅 [07 - 分发](../../07-分发/Electron-Forge概述.md)。
