# 基本要求

> 官方文档：[基本要求 | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/tutorial-prerequisites)

这是 Electron 教程的第 1 部分。

## 目标

本教程首先指导你完成从头开始拼凑一个最小的 Electron 应用程序的过程，然后教你怎么使用 Electron Forge 打包并将其分发给用户。

如果希望从单命令样板开始项目，建议使用 Electron Forge 的 `create-electron-app` 命令。

## 前言

Electron 是网页应用的原生包装层，在 Node.js 环境中运行。本教程假设你已经对 Node.js 和前端 Web 开发有一定了解。推荐先阅读：

- [Web 入门 (MDN)](https://developer.mozilla.org/zh-CN/docs/Learn)
- [Node.js 官方入门指南](https://nodejs.org/zh-cn/learn/getting-started/introduction-to-nodejs)

## 所需工具

### 代码编辑器

推荐使用 Visual Studio Code，也可以选择任何喜欢的文本编辑器。

### 命令行工具

- **Windows**：命令提示符 / PowerShell
- **macOS**：Terminal
- **Linux**：视发行版决定

### Git 和 GitHub

虽然不是必需的，但后续教程会使用 GitHub Releases 实现自动更新。建议：

- 创建 GitHub 账号
- 安装 Git
- 创建本地 Git 仓库并在完成各步骤后提交代码

### Node.js 和 npm

需要安装 Node.js 和 npm，推荐安装最新的 LTS 版本。

```sh
node -v
npm -v
```

> **注意**：开发环境需要 Node.js，但 Electron 应用运行时使用内置的 Node.js 运行时，终端用户不需要安装 Node.js。

查看应用内置的 Node.js 版本，可访问主进程或预加载脚本中的 `process.versions` 变量。

### Windows 注意事项

如果使用 Windows 系统，在本教程中请不要使用 WSL（Windows Subsystem for Linux），否则在尝试运行应用时可能会遇到问题。
