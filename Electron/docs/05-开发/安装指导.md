# 安装指导

> 官方文档：[安装指导 | Electron](https://www.electronjs.org/zh/docs/latest/tutorial/installation)

## 前置要求

1. 前置安装：Node.js 和 npm
2. 前置知识：HTML、CSS、JS、Node.js
3. **注意**：避免使用 WSL（Windows Subsystem for Linux）

## 安装过程

```sh
npm init
npm install electron --save-dev
```

## 运行 Electron ad-hoc

```sh
npx electron .
```

通过此方式在当前目录下运行 Electron，但应用的依赖不会被安装。

## 代理

需要设置环境变量 `ELECTRON_GET_USE_PROXY` 为任何值，附加环境变量取决于 Node 版本：

- [Node 10 以上](https://github.com/gajus/global-agent/blob/v2.1.5/README.md#environment-variables)
- [Node 10 之前](https://github.com/np-maintain/global-tunnel/blob/v2.7.1/README.md#auto-config)

## 自定义镜像和缓存

### 镜像

```sh
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
ELECTRON_CUSTOM_DIR="{{ version }}"
```

URL 组成：`ELECTRON_MIRROR + ELECTRON_CUSTOM_DIR + '/' + ELECTRON_CUSTOM_FILENAME`

### 缓存

| 平台 | 缓存路径 |
|------|----------|
| Linux | `$XDG_CACHE_HOME` 或 `~/.cache/electron/` |
| macOS | `~/Library/Caches/electron/` |
| Windows | `$LOCALAPPDATA/electron/Cache` 或 `~/AppData/Local/electron/Cache/` |

## 跳过二进制包下载

设置 `ELECTRON_SKIP_BINARY_DOWNLOAD` 环境变量阻止二进制文件下载，适用于 CI 环境中的单元测试：

```sh
ELECTRON_SKIP_BINARY_DOWNLOAD=1 npm install
```

## 故障排查

常见网络错误：`ELIFECYCLE`、`EAI_AGAIN`、`ECONNRESET`、`ETIMEDOUT`

- 尝试切换网络或稍后重试
- 从 [electron/electron/releases](https://github.com/electron/electron/releases) 直接下载
- `EACCESS` 错误：修复 npm 权限
- 慢网络：使用 `--verbose` 标志显示下载进度
- 强制重新下载：设置 `force_no_cache=true`

```sh
sudo npm install electron --unsafe-perm=true
npm install --verbose electron
```
