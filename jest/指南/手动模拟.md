# 手动模拟

参考文档: [手动模拟](https://jest.nodejs.cn/docs/manual-mocks)

对应实现见 `jest/models/`、`jest/manual-mock.test.js`。

## 作用

用假数据替代真实模块（网络、数据库等），让测试快且稳定。

## 用户模块

在模块旁建 `__mocks__/`，放入同名文件：

```
models/
  user.js
  __mocks__/
    user.js
```

需要时显式：`jest.mock('./models/user')`。

## Node 模块

对 `lodash` 等，把模拟放在项目根旁 `__mocks__/`（与 `node_modules` 同级），一般会自动启用；作用域包用 `__mocks__/@scope/name.js`。

## 注意

* `jest.mock` 会被提升到文件顶部（在 import 之前）
* `automock: true` 时，有手动模拟也会自动用；要用真模块则 `jest.unmock`
* 可用 `jest.createMockFromModule` 基于自动模拟再覆盖部分实现
