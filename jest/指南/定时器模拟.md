# 定时器模拟

参考文档: [定时器模拟](https://jest.nodejs.cn/docs/timer-mocks)

对应实现见 `jest/timer.js`、`jest/timer.test.js`。

## 为何需要

`setTimeout` / `setInterval` 依赖真实时间，测试会变慢且不稳定。Jest 可用假定时器控制时间流逝。

```js
jest.useFakeTimers();
// ...
jest.useRealTimers(); // 恢复
```

## 常用 API

| API | 作用 |
|-----|------|
| `jest.runAllTimers()` | 执行所有排队定时器 |
| `jest.runOnlyPendingTimers()` | 只跑当前已挂起的（避免无限递归） |
| `jest.advanceTimersByTime(ms)` | 时间前进 ms，执行该窗口内任务 |
| `jest.clearAllTimers()` | 清空待执行定时器 |

## 选择性伪造

`jest.useFakeTimers({ doNotFake: ['performance'] })` 可保留部分原生 API。
