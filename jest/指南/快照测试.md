# 快照测试

参考文档: [快照测试](https://jest.nodejs.cn/docs/snapshot-testing)

对应实现见 `jest/snapshot.test.js`。

## 作用

把可序列化的输出存成快照；下次运行再对比，发现意外变更。不限于 React，任意可序列化值都可以。

```js
expect(data).toMatchSnapshot();
```

## 更新快照

有意改了输出后：

```bash
jest --updateSnapshot
# 或 -u
```

监视模式下也可进入交互式快照模式，逐个确认更新。

## 内联快照

`toMatchInlineSnapshot()` 把快照写回源码，无需单独 `.snap` 文件。

## 属性匹配器

对不稳定字段（如 id、时间戳）用非对称匹配器，再写入快照：

```js
expect(user).toMatchSnapshot({
  createdAt: expect.any(Date),
  id: expect.any(Number),
});
```

## 注意

* 快照应随代码一起提交并参与 Code Review
* 先修真实 bug，再更新快照，避免把错误行为固化进去
