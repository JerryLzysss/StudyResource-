# 入门

参考文档: [入门](https://jest.nodejs.cn/docs/getting-started)

## 安装

```bash
npm install --save-dev jest
```

也可用 `yarn add --dev jest` 或 `pnpm add --save-dev jest`。

## 第一个测试

`sum.js`：

```js
function sum(a, b) {
  return a + b;
}
module.exports = sum;
```

`sum.test.js`：

```js
const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});
```

`package.json` 中增加：

```json
{
  "scripts": {
    "test": "jest"
  }
}
```

运行 `npm test`，看到 `PASS` 即成功。该例用 `expect` + `toBe` 做完全相等断言。

对应实现见 `jest/sum.js`、`jest/sum.test.js`。

## 从命令行运行

全局安装后可直接：

```bash
jest my-test --notify --config=config.json
```

更多选项见 [Jest CLI](https://jest.nodejs.cn/docs/cli)。

## 附加配置

生成基础配置：

```bash
npm init jest@latest
```

### 使用 Babel

```bash
npm install --save-dev babel-jest @babel/core @babel/preset-env
```

`babel.config.js`：

```js
module.exports = {
  presets: [['@babel/preset-env', { targets: { node: 'current' } }]],
};
```

Jest 未设置时会把 `NODE_ENV` 设为 `'test'`，可用 `api.env('test')` 做条件配置。

安装 Jest 时会带上 `babel-jest`；若不想自动转译，可在 `jest.config.js` 中设 `transform: {}`。

### 与打包器一起使用

| 工具 | 说明 |
|------|------|
| Webpack | 可用，资源/别名等需额外配置，见指南 |
| Vite | 不支持 Jest（插件系统不兼容），可改用 Vitest |
| Parcel | 零配置即可配合使用 |

### 使用 TypeScript

- **babel**：装 `@babel/preset-typescript`，仅转译、不做类型检查
- **ts-jest**：带 source map 的 TS 预处理器，可做类型相关支持
- **类型**：`@jest/globals`（官方、随 Jest 更新）或 `@types/jest`（全局类型、第三方维护）

### 使用 ESLint

从 `@jest/globals` 导入 `describe`/`it` 等，或配置 `globals.jest` / `eslint-plugin-jest`，避免 `no-undef`。
