# 使用匹配器

参考文档: [使用匹配器](https://jest.nodejs.cn/docs/using-matchers)

通过匹配器对值做不同类型的断言。对应实现见 `jest/match.test.js`。

## 相等

* `toBe`：完全相等，底层用 `Object.is`（含类型）
* `toEqual`：递归比较对象/数组字段
* `toStrictEqual`：比 `toEqual` 更严格（不忽略 `undefined`、数组空洞、对象原型差异等）

注：`toEqual` 会忽略 `undefined`、数组空缺或对象类型不匹配的部分差异。

## 真假 / 空值

* `toBeNull` / `toBeUndefined` / `toBeDefined`
* `toBeTruthy` / `toBeFalsy`

## 否定

`not` 可加在匹配器前取反，如 `expect(n).not.toEqual(0)`。

## 数字

* `toBeGreaterThan` / `toBeGreaterThanOrEqual`
* `toBeLessThan` / `toBeLessThanOrEqual`
* `toBeCloseTo`：浮点近似比较（避免 `0.1 + 0.2` 精度问题）

## 字符串 / 数组

* `toMatch`：正则匹配字符串
* `toContain`：可迭代对象是否包含某项

## 异常

* `toThrow`：函数调用时是否抛错（需传函数，不要直接调用）

更多匹配器见 [expect API](https://jest.nodejs.cn/docs/expect)。
