17.4.1 使用复制进行备份_MySQL 8.0 参考手册

- Skip to Main Content Documentation Section Menu:  MySQL 8.0 参考手册 MySQL 8.0 参考手册 / 第十七章复制 / 17.4 复制解决方案 / 17.4.1 使用复制进行备份 要将复制用作备份解决方案，请将数据从源复制到副本，然后备份副本。副本可以暂停和关闭，而不会影响源的运行操作，因此您可以生成 “实时”数据的有效快照，否则需要关闭源。 备份数据库的方式取决于它的大小，以及您是仅备份数据，还是备份数据和副本状态，以便在发生故障时重建副本。因此有两种选择： 如果您正在使用复制作为使您能够备份源上的数据的解决方案，并且您的数据库的大小不是太大，则mysqldump工具可能是合适的。请参阅 第 17.4.1.1 节，“使用 mysqldump 备份副本”。
- 对于较大的数据库，如果mysqldump不切实际或效率低下，您可以改为备份原始数据文件。使用原始数据文件选项还意味着您可以备份二进制和中继日志，以便在副本出现故障时重新创建副本。有关详细信息，请参阅 第 17.4.1.2 节，“从副本备份原始数据”。

      另一种可用于源服务器或副本服务器的备份策略是将服务器置于只读状态。备份是针对只读服务器执行的，然后将其更改回其通常的读/写操作状态。请参见
      第 17.4.1.3 节，“通过将其设置为只读来备份源或副本”。
